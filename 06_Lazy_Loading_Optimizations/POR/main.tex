% Abstract
\begin{abstract}
Frameworks de validacao de modelos ML frequentemente carregam multiplos modelos e datasets na memoria simultaneamente, resultando em overhead significativo de tempo (60-120s) e memoria (10-50GB) para experimentos que testam apenas subconjuntos das validacoes disponiveis. Apresentamos estrategias de lazy loading que adiam o carregamento de recursos ate que sejam efetivamente necessarios, combinadas com intelligent caching para reutilizacao. Nossa implementacao no DeepBridge (1) carrega modelos sob demanda apenas quando testes especificos os requerem, (2) implementa cache LRU para predicoes reutilizaveis, (3) usa weak references para permitir garbage collection automatico, e (4) paraleliza carregamento de recursos independentes. Benchmarks em 50 experimentos reais mostram: \textbf{30-50s economia} em tempo de setup (reducao de 45-60\%), \textbf{-42\% uso de memoria} via lazy loading, \textbf{-30\% tempo total} via cache de predicoes, e \textbf{zero overhead} quando todos testes sao executados (lazy = eager neste caso). Ablation study confirma que cada componente contribui significativamente. Framework permite usuarios executarem subconjuntos de validacoes de forma eficiente sem penalidade de performance.
\end{abstract}
