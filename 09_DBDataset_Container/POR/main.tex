% Abstract
\begin{abstract}
Validacao abrangente de modelos de Machine Learning requer gestao coordenada de dados de treino/teste, features, targets, modelos, e predicoes---mas ecosistema ML atual fragmenta esses elementos em objetos distintos (DataFrames, arrays, dicionarios), criando overhead de integracao e erro propenso. Apresentamos \textbf{DBDataset}, um container de dados unificado com \textbf{inferencia automatica de features} que simplifica validacao de modelos ao: (1) encapsular todos os elementos de validacao em interface unica, (2) detectar automaticamente tipos de features (categoricas/numericas) sem configuracao manual, (3) suportar multiplos workflows de inicializacao (dados unificados, pre-separados, com modelo, com probabilidades), (4) integrar-se seamlessly com suites de validacao (robustness, uncertainty, fairness, resilience). Nossa implementacao no DeepBridge suporta \textbf{4 modos de inicializacao}, \textbf{conversao automatica} entre DataFrame/array/scikit-learn Bunch, e \textbf{integracao nativa} com 6 validation suites. Avaliacao empirica em \textbf{3 case studies} (classificacao binaria, regressao, multi-class) demonstra: reducao de \textbf{73\%} em linhas de codigo para setup de validacao (68 LOC $\rightarrow$ 18 LOC), \textbf{100\% de acuracia} em inferencia automatica de features em datasets UCI/Kaggle, e \textbf{compatibilidade completa} com pipelines scikit-learn, XGBoost, LightGBM. DBDataset estabelece novo padrao para reproducibilidade e simplicidade em validacao de modelos ML.
\end{abstract}
